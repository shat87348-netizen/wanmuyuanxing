# 用户角色与场景

> 本系统是**数据处理子系统的后台管理界面**，定位是**后台处理任务的状态查看与配置管理**。
> 不是值班指控类系统：**无告警铃声 / 无大屏 / 无快捷键 / 无 4K**。
>
> 用户只有 2 种角色。

---

## 角色 1 — 管理员（admin）

**职责**：系统全权限。

**能做**：
- 配置采集接口（新增 / 修改 / 删除接口实例）
- 启停采集接口（启用 / 禁用 / 重启 / 熔断 / 解除熔断）
- 限速 / 流量控制
- 配置参数解析模型（位域 / 公式 / 阈值 / 校准）
- 解析模型 ZIP 导入 / 导出
- 创建 / 编辑 / 激活 / 删除 试验任务
- 查看所有状态、日志、告警
- 删除 agent 注册条目

**典型操作流**：
1. 进系统 → 选择当前试验任务
2. 配置该任务的解析模型（或导入 ZIP）
3. 在采集代理 & 接口页启动相关接口
4. 监控后台处理任务的运行状态
5. 出问题时查日志 / 调阈值 / 调校准算法

---

## 角色 2 — 普通用户（user）

**职责**：只读查看。

**能做**：
- 查询所有数据（遥测、告警、处理结果）
- 查看后台处理任务的状态（吞吐 / 健康 / 队列深度）
- 查看采集代理 & 接口的连接状态、流量统计
- 查看解析模型配置（只读，不能改）
- 查看日志
- 查看试验任务列表 + 切换查看视角（不能创建 / 激活 / 删除）

**不能做**：
- 任何"写"操作：启停接口、改配置、删任务、导入模型……

**典型操作流**：
1. 进系统 → 选择当前查看的试验任务
2. 看后台处理状态总览（绿 / 黄 / 红）
3. 看告警列表
4. 查具体参数的解析结果 / 历史曲线

---

## 角色权限矩阵（设计 / 实现指导）

| 功能 | admin | user |
|---|---|---|
| 总览 / 状态查看 | ✅ | ✅ |
| 试验任务 列表 / 详情 | ✅ | ✅ |
| 试验任务 创建 / 激活 / 删除 | ✅ | ❌ |
| 当前任务上下文切换（仅查看视角） | ✅ | ✅ |
| 采集代理 / 接口 状态查看 | ✅ | ✅ |
| 采集接口 启用 / 禁用 / 限速 / 熔断 / 删除 | ✅ | ❌ |
| 解析模型 查看 | ✅ | ✅ |
| 解析模型 编辑 / 保存 / 导入 / 导出 | ✅ | ❌ |
| 校准算法 配置 | ✅ | ❌ |
| 告警 查看 | ✅ | ✅ |
| 告警 处理 / 删除 | ✅ | ❌ |
| 日志 查看 | ✅ | ✅ |

**前端实现指南**：
- 用 `useCurrentUser()` composable 拿当前角色
- 写操作按钮：根据角色显隐（不仅是 disable，**完全不渲染**给 user 更干净）
- 写操作 API：后端也要做服务端校验（前端只是 UX 层）
- 现阶段后端用 MockAdminFilter 固定 admin；前端先用 localStorage 切换角色用于开发联调，待后端接入登录后切换

## 共通设计约束

| 项 | 约束 |
|---|---|
| 屏幕 | 1920×1080（默认） / 1366×768（兼容） |
| 主题 | 已实现的 6 主题切换（军绿 / 风格1 / 风格2 × 浅深），不动 |
| 字体 | 中文系统默认；数字等宽（无需引入自定义字体） |
| 国际化 | 中文 only |
| 性能 | 列表轮询 ≤ 10s；告警 WebSocket 推送即可，不需要响铃 |
| 弱网 | WebSocket 断线静默重连，右下角小指示符；不需要弹窗骚扰 |
| 错误 | API 失败 → Toast + 重试按钮 |
| 容错 | admin 的破坏性操作（删任务、删模型）二次确认即可，无需复杂审批 |

---

## 反模式（不要做）

- ❌ **告警铃声 / 闪烁 / 大字号弹屏** — 不是值班指控系统
- ❌ **大屏拼接 / 4K 适配** — 默认 PC 即可
- ❌ **快捷键** — 后台管理类不需要
- ❌ **复杂的"我的工作台" / 角色定制工作流** — 只有 admin/user 两角色
- ❌ Demo 风：硬编码数据永远不变
- ❌ 列表只读：所有列表必须支持搜索 + 排序 + 分页
- ❌ 假数据：空状态不要伪造数据；用"暂无"占位
- ❌ 玻璃拟态 / 全屏视频背景 / 渐变装饰
